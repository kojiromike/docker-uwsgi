What's next?

Run your own Python application! You can create a new image based on this one, or you can dynamically update this container and run your application directly.

Make your own image

In the test image, we install werkzeug as a test application.


    $ docker build --tag werkzeug_testapp - << DOCKERFILE
    FROM uwsgi:latest
    RUN pip install werkzeug
    ENV UWSGI_MODULE=werkzeug.testapp:test_app
    DOCKERFILE
    $ docker run --detach --name werkzeug_testapp -p 8080:8080 werkzeug_testapp

Making your own image is better if you want a reproducible static image that starts immediately with your application.

Dynamically update this container

Install an application directly in this container, then update the uwsgi.ini to point to it. For example:


    $ docker run --detach --publish-all --env PYPI_PACKAGES=werkzeug uwsgi:latest -w werkzeug.testapp:test_app

Dynamic reloading is useful if you are tweaking settings and trying things out.
